## 1. 执行摘要

### 1.1 项目目标
构建自动化盘前资金流向分析系统，每日开盘前生成市场资金流向报告，识别主力板块和个股，为买入决策提供数据支持。

### 1.2 核心价值
| 价值点 | 说明 |
|--------|------|
| **时效性** | 每日9:00自动生成报告，抢占先机 |
| **客观性** | 基于全市场资金流向数据，减少主观判断 |
| **系统性** | 多维度分析（板块+个股+资金），全面覆盖 |
| **可追溯** | 历史数据存储，支持策略回测 |

### 1.3 技术可行性
- **数据源**: AKShare免费接口，覆盖A股全市场
- **部署环境**: 阿里云VPS（已部署AKShare + Jupyter）
- **开发周期**: 预计32工时（4个工作日）

## 2. 项目背景

### 2.1 需求分析
当前手工查看资金流向存在以下痛点：
- 信息分散：需同时查看多个APP/网站
- 时效性差：错过早盘黄金时间
- 缺乏量化：主观判断难以验证
- 无法回溯：历史决策无法复盘

### 2.2 解决方案
通过自动化系统实现：
1. **数据聚合**: 统一拉取行情、资金、板块数据
2. **智能评分**: 多因子模型量化板块强度
3. **策略选股**: 基于动量/反转/资金流筛选标的
4. **定时报告**: 每日自动生成分析报告

## 6. 关键算法设计

### 6.1 板块强度评分算法

**评分维度** (总分100分)

| 维度 | 权重 | 评分规则 |
|------|------|----------|
| 涨跌幅 | 25% | ≥5%得25分，3-5%得20分，1-3%得15分，0-1%得10分，负值得0-5分 |
| 资金流入 | 25% | 净流入占比≥5%得25分，按比例递减 |
| 上涨广度 | 25% | 上涨家数占比×25 |
| 龙头股 | 25% | 龙头股涨幅≥7%得25分，按比例递减 |

**评分示例**:
```
板块: 机器人
涨跌幅: +3.2% → 20分
资金流入占比: +4.5% → 20分
上涨家数占比: 70% → 17.5分
龙头股涨幅: +5.8% → 20分
总分: 77.5分 (强势板块)
```

### 6.2 多因子选股模型

#### 策略一: 动量策略 (追涨)

**选股条件**:
- 涨幅 2% ~ 7% (避免涨停无法买入)
- 量比 > 150% (放量)
- 成交额 > 3亿 (流动性充足)
- 1日主力净流入 > 0

**评分公式**:
```
Score = 涨幅×0.4 + (量比-100)×0.1 + 资金流入×0.4
```

#### 策略二: 反转策略 (抄底)

**选股条件**:
- 跌幅 -7% ~ -3% (超跌但非跌停)
- 成交额 > 2亿
- 5日主力净流入 > 0 (长期资金仍在)

**评分公式**:
```
Score = |跌幅|×0.3 + 5日资金流入×0.5 + 成交额×0.2
```

#### 策略三: 资金流策略 (跟随主力)

**选股条件**:
- 5日主力持续净流入
- 今日净流入 > 1亿
- 涨幅 < 7% (避免追高)

**评分公式**:
```
Score = 5日资金流入×0.6 + 1日资金流入×0.3 + (5-涨幅)×0.1
```
